# RAG-Py Context

This file defines the shared domain language for this project.
Use these terms consistently in code, docs, PRs, and AI prompts.

## Core Domain Terms

- **Knowledge Base (KB)**: A logical collection of documents with retrieval settings.
- **Document**: A user-uploaded file (TXT/MD/PDF/DOCX) attached to one KB.
- **Chunk**: A split text segment derived from a document for embedding.
- **Embedding**: Vector representation generated by the embedding provider (BGE by default).
- **Vector Index**: Vector store entries used for semantic retrieval.
- **Query**: User question scoped to one KB.
- **Source Citation**: Structured evidence returned with answer (document, chunk, score).

## Pipeline Terms

- **Ingestion**: Upload -> parse -> split -> embed -> vector upsert.
- **Retrieval**: Search vector index by query embedding (with optional filters/rerank).
- **Generation**: LLM synthesis from retrieved context (MiMo by default, config-driven).
- **Indexing Job**: Async worker task that builds/rebuilds vectors for a document.

## Status Terms

- **uploaded**: File accepted and stored.
- **processing**: Parsing/splitting/embedding in progress.
- **indexed**: Searchable and ready for query.
- **failed**: Indexing failed; see error code and logs.

## Design Rules

- PostgreSQL is the metadata source of truth.
- Vector store is rebuildable from metadata + source files.
- Provider selection (BGE, MiMo) is configuration-driven, never hardcoded in business logic.
- Every answer should include citations whenever context is found.
